# Lab 12 — Помещения и стеллажи (вариант 3)

Консольное приложение и HTTP API на **FastAPI** + **SQLAlchemy 2.0** + **Alembic** + **PostgreSQL**. В интерфейсе не используются суррогатные ключи БД: только порядковый номер строки на экране и понятные поля (название помещения, номер стеллажа).

## Настройка

1. Скопируйте и отредактируйте [`config.yaml`](config.yaml) (хост, БД, пользователь, пароль).
2. Из **корня проекта лабораторной** (эта папка, где лежат `pyproject.toml` и `alembic.ini`):

   ```bash
   cd labs/lab12
   uv sync
   ```

3. Миграции применяются при старте CLI/API; при необходимости вручную:

   ```bash
   uv run alembic upgrade head
   ```

## Консоль

```bash
cd labs/lab12
uv run lab12-cli
```

Доступны постраничный просмотр помещений и стеллажей, добавление / изменение / удаление помещений и стеллажей.

## HTTP API

Запускайте **из папки `labs/lab12`**, чтобы пакет `lab12` находился в окружении `uv`:

```bash
cd labs/lab12
uv run uvicorn lab12.api.app:app --reload --host 127.0.0.1 --port 8000
```

После запуска откройте **http://127.0.0.1:8000/** — главная («Управление складом»), разделы **http://127.0.0.1:8000/warehouse/rooms** и **http://127.0.0.1:8000/warehouse/racks** (статика в [`site/`](site/)). Сводный список стеллажей по складу: `GET /racks?skip=&limit=&sort=&order=&room_name=`. Документация API: http://127.0.0.1:8000/docs

Если выполнить `uvicorn` из родительского каталога репозитория без указания модуля, импорт `server` или `lab12` может завершиться ошибкой — всегда используйте каталог лабораторной или установленный пакет.

Список помещений с пагинацией: `GET /rooms?skip=0&limit=50`. Стеллажи помещения: `GET /rooms/{room_name}/racks?skip=0&limit=50`. Идентификаторы в URL — только **название помещения** и **номер стеллажа**, не `id` из БД.
