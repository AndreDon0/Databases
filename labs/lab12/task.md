## Общая часть задания

1. Загрузите эталонный проект и настройте его для вашего сервера и пользователя СУБД, а также установите нужные библиотеки для Python.

2. Замените базовые таблицы `people` и `phone` в проекте на таблицы, указанные в вашем варианте, доработав все проверки целостности данных и соответствия типам данных. Не должно при добавлении и других действиях происходить ошибок PostgreSQL, приводящих к «вылету» программы или к поведению без нормальных объяснений. Сообщения PostgreSQL о несоответствии типов, срабатывании ограничений целостности или других аспектах таблиц **не** считаются нормальными объяснениями для пользователя. Интерфейс ввода **не должен** использовать суррогатных ключей — только понятные людям нетехнические данные или порядковый номер записи на экране.

3. Доработайте для **первой** таблицы возможности **удаления** и **редактирования** сущностей (должны работать добавление, изменение, удаление с учётом связанных объектов **второй** таблицы). Интерфейс не должен использовать суррогатных ключей — только понятные людям нетехнические данные или порядковый номер записи на экране.

4. Доработайте для **второй** таблицы: просмотр списка сущностей по указанному значению ключа из первой таблицы; добавление новых сущностей с этим ключом (включая все проверки целостности и типов); удаление сущностей в том же интерфейсе. Интерфейс не должен использовать суррогатных ключей — только понятные людям нетехнические данные или порядковый номер записи на экране.

5. Устраните все потенциальные **SQL-injection** уязвимости.

**Важно:** нигде в программе пользователь **не должен** вводить значения суррогатных ключей (например, значения поля `id`).

Если сущности из задания у вас реализованы большим числом таблиц, разделите интерфейс на два шага: основные таблицы и связанные.

**Задания 6 и 7** не входят в обязательный объём лабораторной работы; они для саморазвития. По желанию преподавателя за их выполнение могут быть начислены дополнительные баллы.

6. Сделайте **постраничный** просмотр сущностей.

7. Для второй таблицы в интерфейсе просмотра списка по ключу из первой таблицы добавьте **изменение** сущностей второй таблицы (`UPDATE`).
